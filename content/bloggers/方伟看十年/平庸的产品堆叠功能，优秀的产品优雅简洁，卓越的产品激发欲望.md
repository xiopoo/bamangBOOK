# 平庸的产品堆叠功能，优秀的产品优雅简洁，卓越的产品激发欲望

> 方伟Rey · 方伟看十年 · 2022-10-13 11:25


以前做产品有一条很简单的判断原则：**能否“激发欲望，消除恐惧”（**背后是更精细的模型，比如AIDMA法则——Attention 注意-Interest 兴趣 - Desire 欲望 - Memory 记忆- Action 行动）。

![](./assets/057a16170ac30a813c2ebcab95d8d445.png)

今天体验了一款我评价标准里的卓越产品——成家相亲。他在每个环节总能激起用户的欲望，让用户看到希望、看到可能性，甚至心甘情愿为这种可能性买单。

![](./assets/5505fe1d60958bd7623743f16113de76.png)

我打开登录页的时候就知道这款产品不简单，**卓越产品是那种光看名字就能打动人的产品，他能实现核心价值主张无损地传播**，比如Boss直聘、今日头条、WiFi万能钥匙、大众点评、淘宝，这款产品的名字就是如此：成家相亲，“成家”是最能touch到帮忙相亲的父母的一个点，直接用在了产品名。

次一点的产品虽然名字无法打动人，但slogan很打动人，比如“雪球，**优秀的投资者**都在这里”、“36kr，让一部分人先**看到未来**”、“飞书，**先进团队**，先用飞书”、“瓜子二手车直买网，没有中间商**赚差价**”，再来看成家相亲的，“父母帮孩子找对象”、“招女婿找儿媳，先和亲家聊”，同样满足。

**评价产品最重要的其实是看他那个“1”对不对，这个“1”指的是用户的核心痛点及其对应的核心价值主张**，如果“1”不对，其他的部分再怎么美轮美奂，都没有意义，比如小鹅拼拼、抖音盒子，他们家的设计真的是很好看，但他始终讲不明白他那个“1”是什么，腾讯这些年推出的新产品，普遍存在这个问题，即设计非常精美，UI交互的水平越来越高，但就是不知道解决是什么问题，有什么增量价值。

成家相亲这款产品的“1”是非常准的（我们不能以过去十年移动互联网的巨鲸公司去要求他，张口就是能不能达到千亿规模、百亿规模，要知道他是一家成立于2020年的公司，这个阶段成立的移动互联网公司，阶段性做成的应该屈指可数，你要用做生意赚钱的逻辑去看他），这是深刻洞察中国婚姻文化的产物，即**中国的父母想让孩子成家立业的欲望比孩子本身还要强烈，更重要的是他们有时间，这也是为数不多能让这个群体真金白银掏钱的需求。**

我猜测他的需求来源是各个城市的公园相亲角，这里面有一个很有意思的结论，即**那些在线下存在了很多年的事物，往往就是满足了很底层的需求才存在到今天的**，比如餐馆、旅馆、青楼、赌坊、寺庙....如果能够使用移动互联网的技术（比如连接）实现线上化的话，PMF的概率是很高的。

“1”确定靠谱后我们看核心流程、功能与设计，你会发现他始终在给父母们兜售希望：

![](./assets/72f9e53e430862fded8704d8b08d6847.png)

跟一个社交创业多年的朋友聊，对这款产品的判断是“投放模型逻辑”，即成本100块一个注册用户，转化的付费产品以488元/3个月的会员为主，也就是说他每4个注册用户需要有1个付费才能把ROI模型跑正，这并不简单。能否快速地让用户感知到核心价值，并且转化成付费用户就决定了这款产品的生死。

![](./assets/f16f9e00a76c67d70bf1470c23dbea62.png)

我们可以来看他怎么实现这个业务目标，打开App的瞬间就很“拼多多”，先送1天的会员，接着我们看第二个页面，他是刻意设计的，顶部是成功相亲的案例，进一步建立信任，让人看到成功的希望；下面推荐给新用户的相亲对象也是刻意设计的，会明显发现“特别优秀”，能看出前面几个相亲对象纯粹是用来激发欲望的，不是真实的，这在以“投放模型逻辑”跑的产品里这样的设计也无可厚非。再看第三个页面，付费点有两个，其一是查看对方孩子的照片，其二是联系对方，这两个付费点都抓得很准，如果你只对联系对方设置为付费点，那么你的会员卖得不会很好，父母们也是颜控，光看照片就筛选掉了不少人，连联系对方都无法操作。

我们来看整个流程，**会发现他做得很轻、很尖、很准，没有任何多余的动作**，只提供联系电话，不交付任何其他重的服务，这是一流产品经理的特征，也有成本收益的思维，懂得在488会员的收益下应该交付什么样的服务，做到超预期即可。正如别人问黄峥，人家说你拼多多的商品质量很差，一箱芒果一半都是烂的，结果他回答，花9.9就买了一箱芒果，在你看来是居然有一半是烂的，但在他看来是还有一半能吃，超值，复购率会告诉我们答案。

这款产品细节的点还有很多，比如下图，有一个“结婚返款”的设计，目的其一是增加信任，会让用户感觉这款产品是真心想帮助我们的孩子结婚的，其二是满足用户占便宜的心理实现更高客单会员的交付。黄峥说过一句很耐人寻味的话，在中国，便宜不是关键，关键是满足让用户占便宜的感觉。

![](./assets/fa93d77da13842ec7d3fe286a4200a9a.png)

成家相亲这种产品，在很多互联网产品经理人眼里，是很low的，甚至不符合他们的价值观，我无意说服他们，只是我们可以从产品设计的角度，而不是道德评判的角度去粗取精，欣赏他们做得好的点，为我们所用。

为什么“激发欲望”很重要，这里其实也有一个科学的模型，叫做Fogg福格模型，即影响人的行为有三个因素，**其一是动机**，也就是欲望，欲望越高，人们做这件事情的动力也就越足；**其二是能力**，即门槛得低，做一件事情所需要的能力越低，能完成的人也就越多；**其三是触发**，光有动机与能力还不行，还得在各个地方提醒他，因为人是很健忘的，你不告诉他，他就真的忘记了。

回到文章的标题：**平庸的产品堆叠功能，优秀的产品优雅简洁，卓越的产品激发欲望**，堆叠功能是最多人做得事情，印象笔记、钉钉、支付宝、百度很多产品都存在这个问题，比如下图是印象笔记的Read it later产品“收藏家”，无法理解他们家产品经理是怎么思考问题的，为什么要往一款Read it later产品里丢发现频道，难道我需要在里面浏览新闻吗？里面的内容是否有区别于其他的社区产品有不得不在这里消费的稀缺性？

![](./assets/04785ec4e6def8b6a5050f35e1a8dd72.png)

把一款产品做到优雅简洁，是Fogg模型中的降低能力门槛，我们看成家相亲，其实他在这个环节也做得足够好，要知道他们的用户群可都是50岁+的中老年人，任何需要思考的点他们都不会操作，你必须做到极简才能让他们用起来。

成家相亲这个团队厉害的地方就在于他们不仅能做到优雅简洁，还能在每个环节都激发用户的欲望，让他们感知到这不是在学习怎么玩一款产品，这多没意思呀，没有人喜欢学习，而是感知到我进入了全国最大的相亲角，我一定要为自己的孩子挑个好对象。
