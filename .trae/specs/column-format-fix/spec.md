# 分栏格式检查与修正 Spec

## Overview
- **Summary**: 对文档内容进行分栏格式检查与修正，重点处理表格中的分栏错位问题，确保"麻省投资信托"等条目作为完整条目呈现，消除空栏问题。
- **Purpose**: 修复表格中数据错位、空栏、内容割裂等问题，确保表格格式统一规范，内容排布合理。
- **Target Users**: 文档读者和网站用户

## Goals
- 修正"麻省投资信托"的分栏错误，确保其作为完整条目呈现
- 消除当前存在的两个空栏问题（道指和LP合伙人列）
- 对所有文档进行系统性分栏检查，识别并修正所有分栏错位问题
- 确保修正后的分栏格式统一规范，内容排布合理
- 完成后进行整体校对，确认所有分栏问题均已解决

## Non-Goals (Out of Scope)
- 修改文档内容的语义或含义
- 添加新的文档或内容
- 修改网站框架或结构

## Background & Context
- 现有文档通过PDF提取生成，存在表格分栏识别错误问题
- 从用户提供的截图来看，表格中"道指"和"LP合伙人"两列是空的，数据只有6列，而表头有8列
- 这是一个典型的分栏错位问题，需要将表头调整为6列，或者将数据填充到正确的列中

## Functional Requirements
- **FR-1**: 修正"麻省投资信托"的分栏错误，确保其作为完整条目呈现
- **FR-2**: 消除空栏问题，确保表格列数一致
- **FR-3**: 对所有文档进行系统性分栏检查，识别并修正所有分栏错位问题
- **FR-4**: 确保修正后的分栏格式统一规范，内容排布合理

## Non-Functional Requirements
- **NFR-1**: 修复后的表格应保持原始内容不变，仅调整格式
- **NFR-2**: 修复后的表格应符合Markdown表格标准，列数一致
- **NFR-3**: 修复后的表格数据应准确完整，无遗漏或错误

## Constraints
- **Technical**: 使用现有的Node.js脚本进行修复
- **Dependencies**: 依赖现有的文档文件
- **Output**: 输出到 `/content/partnership/`、`/content/letters/` 等网站目录

## Assumptions
- 现有文档内容基本准确，仅格式和分栏有问题
- PDF提取的原始文本包含足够的信息来修复分栏格式问题

## Acceptance Criteria

### AC-1: 麻省投资信托分栏修正
- **Given**: 文档中"麻省投资信托"存在分栏错误
- **When**: 运行修复脚本
- **Then**: "麻省投资信托"作为完整条目呈现，没有分栏错误
- **Verification**: `human-judgment`

### AC-2: 空栏问题消除
- **Given**: 表格中存在空栏问题（道指和LP合伙人列）
- **When**: 运行修复脚本
- **Then**: 表格中没有空栏，数据完整填充到正确的列中
- **Verification**: `human-judgment`

### AC-3: 分栏错位修复
- **Given**: 文档中存在分栏错位问题
- **When**: 运行修复脚本
- **Then**: 所有分栏错位问题被修复，表格格式正确
- **Verification**: `human-judgment`

### AC-4: 分栏格式统一规范
- **Given**: 修复后的文档
- **When**: 使用Markdown渲染器渲染
- **Then**: 表格能正常渲染，格式正确，无内容截断或不完整现象
- **Verification**: `human-judgment`
