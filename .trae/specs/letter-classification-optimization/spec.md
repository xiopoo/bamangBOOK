# 信件分类优化与格式修正 - 产品需求文档

## Overview
- **Summary**: 对巴菲特信件系统进行全面优化，包括建立独立互斥的分类体系、整合年度大事记内容、修正标题与格式问题。
- **Purpose**: 提升内容浏览的清晰度和逻辑性，确保合伙人信与股东信完全分离，提取并规范化展示年度大事记，修正格式错误。
- **Target Users**: 文档阅读用户、内容维护人员

## Goals
- 建立"纯粹合伙人信"与"股东信"两个独立互斥的分类体系
- 补充1956年合伙人信内容
- 设计"年度大事记"展示区域，系统提取并规范化展示相关内容
- 修正标题分类错误和编号格式问题

## Non-Goals (Out of Scope)
- 不修改前端表格组件的UI样式
- 不改变信件的核心内容
- 不添加新的路由页面（除必要的组件外）

## Background & Context
- 当前股东信分类中混杂了合伙人信内容（1965-1970年）
- 1956年合伙人信缺失，相关内容在articles/巴菲特合伙契约_1956.md中
- 信件中包含年度重要事件记录，但未以独立板块形式呈现
- 部分正文中的"1、2、3"等列表编号被错误地作为标题显示

## Functional Requirements
- **FR-1**: 创建1956年合伙人信文件，补充缺失内容
- **FR-2**: 修改分类逻辑，使合伙人信与股东信完全分离（1965-1970年只显示股东信）
- **FR-3**: 更新股东信列表页，移除"含合伙人信"标签
- **FR-4**: 扫描并提取信件中的年度大事记内容
- **FR-5**: 设计并实现"年度大事记"展示组件
- **FR-6**: 修正编号格式，将"1、2、3"等编号段落与标题区分渲染
- **FR-7**: 审核并修正标题分类错误

## Non-Functional Requirements
- **NFR-1**: 修改不影响现有路由和页面结构
- **NFR-2**: 保持原始文本的准确性
- **NFR-3**: 提升整体内容的结构性和可读性

## Constraints
- **Technical**: 使用现有技术栈（Next.js、React、Markdown）
- **Business**: 修复结果需保持数据的准确性和可读性

## Assumptions
- 1956年合伙人信内容可从articles/巴菲特合伙契约_1956.md提取
- 大事记内容格式多样，需要灵活的提取规则
- 编号格式修正需要在渲染层处理

## Acceptance Criteria

### AC-1: 1956年合伙人信补充
- **Given**: 系统中缺少1956年合伙人信
- **When**: 创建partnership_1956-巴菲特致合伙人信.md文件
- **Then**: /partnership页面显示1956年条目，内容正确
- **Verification**: `programmatic`

### AC-2: 分类体系分离
- **Given**: 当前股东信页面在1965-1970年同时显示合伙人信和股东信
- **When**: 修改分类逻辑
- **Then**: 股东信页面只显示股东信，合伙人信页面只显示合伙人信，两类信件完全分离
- **Verification**: `programmatic`

### AC-3: 年度大事记展示
- **Given**: 信件中包含年度重要事件记录
- **When**: 访问信件详情页面
- **Then**: 页面显示"年度大事记"独立板块，内容从信件中提取并规范化展示
- **Verification**: `human-judgment`

### AC-4: 编号格式修正
- **Given**: 正文中存在"1、2、3"等编号格式
- **When**: 渲染信件内容
- **Then**: 编号段落以标准列表格式呈现，不被识别为标题
- **Verification**: `human-judgment`

### AC-5: 项目构建验证
- **Given**: 所有修改完成
- **When**: 运行npm run build
- **Then**: 项目构建成功，无错误
- **Verification**: `programmatic`
