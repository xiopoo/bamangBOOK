# 巴菲特股东信知识库重建 Spec

## Why
用户希望从零开始构建一个巴菲特股东信知识库，类似维基百科的结构：
- 81封信件（1956-2025）互相关联
- 35个投资概念完整知识卡片
- 61家公司档案
- 7位关键人物档案
- 4,194条交叉链接

## What Changes

### 核心架构
- **左侧导航栏**：分类浏览（合伙人信、股东信、概念、公司、人物）
- **知识卡片**：每个概念/公司/人物都有完整页面
- **交叉链接**：所有[[概念]]、[[公司]]、[[人物]]都可点击跳转

### 知识卡片结构（完整版）
每个概念/公司/人物卡片包含：
1. **定义与起源** - 这个概念最早出现在哪封信
2. **核心要义** - 3-4个子论点
3. **实战案例** - 具体到哪家公司、什么年份、多少钱
4. **常见误解** - 巴菲特本人怎么纠偏的
5. **思想演变** - 从50年代到2020年代的演变轨迹
6. **原话引用** - 5-10条巴菲特的原文

### 页面结构
```
/                    - 首页（统计数据 + 快速入口）
/letters             - 股东信列表
/letters/[year]      - 股东信详情页（带交叉链接）
/partnership         - 合伙人信列表
/partnership/[year]  - 合伙人信详情页
/concepts            - 概念列表
/concepts/[name]     - 概念知识卡片
/companies           - 公司列表
/companies/[name]    - 公司档案
/people              - 人物列表
/people/[name]       - 人物档案
```

## Impact
- Affected code:
  - `src/app/` - 全新页面结构
  - `src/components/` - 知识卡片组件
  - `content/` - 知识库内容结构

## ADDED Requirements

### Requirement: 维基百科式导航
系统 SHALL 提供左侧导航栏，分类展示合伙人信、股东信、概念、公司、人物。

### Requirement: 完整知识卡片
系统 SHALL 为每个概念/公司/人物提供包含6个部分的完整知识卡片。

### Requirement: 交叉链接
系统 SHALL 在信件内容中自动识别并链接[[概念]]、[[公司]]、[[人物]]。

### Requirement: 思想演变时间线
系统 SHALL 展示每个概念在不同年代的演变轨迹。

## 技术规范
- 数据来源：IMA 知识库 API
- 前端：Next.js 14 + Tailwind CSS
- 主题色：橙色 #f97316
