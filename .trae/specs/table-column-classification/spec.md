# 表格栏位分类功能修复 - 产品需求文档

## Overview
- **Summary**: 对系统中所有表格组件的栏位分类功能进行全面检查与修复，确保表格数据能够按照预设的分类规则准确、一致地进行分组展示。
- **Purpose**: 修复文档中存在的表格栏位分类错误，包括列偏移、重复表头、数据错位等问题，确保表格数据正确对应列标题。
- **Target Users**: 文档阅读用户、内容维护人员

## Goals
- 识别并修复所有文档中表格的栏位分类错误
- 分析错误产生的根本原因并优化分类逻辑算法
- 添加必要的数据验证机制
- 确保分类结果符合业务需求和用户预期
- 对所有涉及的表格组件进行全面测试

## Non-Goals (Out of Scope)
- 不修改前端表格组件的UI样式
- 不改变表格的数据结构设计
- 不添加新的表格功能

## Background & Context
- 文档中存在大量Markdown表格，部分表格存在栏位分类错误
- 常见错误类型：年份列后"年"字被分离到下一列导致数据列错位、重复表头、数据单元格内容错位
- 需要开发自动化脚本进行批量检测和修复

## Functional Requirements
- **FR-1**: 检测表格中年份列后"年"字被错误分离的列偏移问题
- **FR-2**: 检测并删除重复的表头行
- **FR-3**: 检测并修复数据单元格内容错位问题（如"小于 0"被分离、多个百分比值被合并）
- **FR-4**: 检测列数不一致问题
- **FR-5**: 自动修复检测到的所有表格错误

## Non-Functional Requirements
- **NFR-1**: 脚本能够处理270+个文档，包含61+个表格
- **NFR-2**: 修复过程不破坏文档的其他内容
- **NFR-3**: 修复后项目能够正常构建

## Constraints
- **Technical**: 使用Node.js开发修复脚本，不引入新的依赖
- **Business**: 修复结果需保持数据的准确性和可读性

## Assumptions
- 表格使用标准Markdown格式
- 表格分隔线包含`|---`或`| ---`模式

## Acceptance Criteria

### AC-1: 列偏移错误检测与修复
- **Given**: 文档中存在年份列后"年"字被分离的表格
- **When**: 运行修复脚本
- **Then**: "年"字被合并到年份列，数据列位置正确
- **Verification**: `programmatic`

### AC-2: 重复表头检测与删除
- **Given**: 文档中存在重复表头行的表格
- **When**: 运行修复脚本
- **Then**: 只保留一个表头行
- **Verification**: `programmatic`

### AC-3: 数据错位修复
- **Given**: 文档中存在数据单元格内容错位的表格
- **When**: 运行修复脚本
- **Then**: 数据内容被正确拆分或合并到对应单元格
- **Verification**: `programmatic`

### AC-4: 项目构建验证
- **Given**: 所有表格错误已修复
- **When**: 运行npm run build
- **Then**: 项目构建成功，无错误
- **Verification**: `programmatic`
