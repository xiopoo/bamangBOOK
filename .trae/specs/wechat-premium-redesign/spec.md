# 公众号文章高级感模板重新设计（大红色主色）

## Why

用户反馈之前的模板设计"一点高级感都没有"。需要从零开始重新设计，以**大红色为主色、黑灰白为辅助色**，打造具有高级感/奢华感的公众号文章排版。目标视觉参考：高端杂志、奢侈品牌排版风格。

## What Changes

- 从零设计全新 CSS，不继承之前任何版本的样式
- 以大红色（如 `#CC0000`）为主色，黑、灰、白为辅助色
- 追求"高级感"——大留白、精致线条、克制用色、高品质字体
- 红色使用要克制且精准，用在关键视觉锚点（标题装饰、分割线、强调标记）
- 黑白灰层次分明：正文近黑、辅助信息灰、背景纯白

## ADDED Requirements

### Requirement: 高级感视觉定位

排版应呈现高端杂志/奢侈品牌的视觉质感。

#### Scenario: 整体视觉印象
- **WHEN** 用户在微信后台打开排版后的文章
- **THEN** 视觉感受应接近高端商业杂志或奢侈品牌排版，而非普通公众号或笔记
- **THEN** 应有明显的精致感和设计感

### Requirement: 大红色主色使用规则

大红色应作为品牌主色使用，但使用克制，只出现在关键视觉位置。

#### Scenario: 红色使用位置
- **WHEN** 用户查看文章大标题
- **THEN** 大标题使用大红色，但不使用任何粗重边框
- **WHEN** 用户查看分割线
- **THEN** 可使用细红线条作为视觉分割
- **WHEN** 用户查看小标题
- **THEN** 小标题使用黑/深灰色，不使用红色
- **WHEN** 用户查看强调内容
- **THEN** 强调内容可使用红色细标记（如小圆点、细下划线），不使用大面积色块

#### Scenario: 红色禁止事项
- **THEN** 任何元素不得使用红色背景色块
- **THEN** 红色不得用于正文文字
- **THEN** 红色不得用于大面积边框或填充

### Requirement: 黑灰白层次

黑白灰应有清晰的层次，支撑高级感。

#### Scenario: 色彩层次
- **WHEN** 用户阅读正文
- **THEN** 正文使用近黑色（`#111` 或 `#1a1a1a`），不加粗
- **WHEN** 用户查看辅助信息
- **THEN** 辅助信息使用中灰色（`#666` 或 `#888`）
- **WHEN** 用户查看分割线和装饰
- **THEN** 分割线使用浅灰（`#e0e0e0` 或更淡）
- **THEN** 背景始终为纯白 `#fff`

### Requirement: 字体与排版高级感

字体和排版是高级感的核心载体。

#### Scenario: 字体选择
- **WHEN** 用户阅读正文
- **THEN** 正文使用衬线体（Georgia + Noto Serif SC），字号 16px，行高 1.9
- **WHEN** 用户查看大标题
- **THEN** 大标题使用无衬线体（PingFang SC / 系统黑体），字号 24px 以上
- **WHEN** 用户查看小标题
- **THEN** 小标题使用无衬线体，字号 17px，字重 600

#### Scenario: 间距与留白
- **THEN** 正文段间距 >= 20px
- **THEN** 两侧留白 >= 24px
- **THEN** 区块间距 >= 30px
- **THEN** 字符间距在标题上适当宽松（`letter-spacing: 1px`）

### Requirement: 装饰与细节

高级感体现在细节装饰上。

#### Scenario: 装饰元素
- **WHEN** 用户查看文章顶部
- **THEN** 标题上方有一条细红装饰线（2px 高，大红色）
- **WHEN** 用户查看引用块
- **THEN** 引用块使用极细深灰左边框（1px）+ 深灰色文字，无背景色块
- **WHEN** 用户查看关键数据/结论
- **THEN** 关键数据/结论可使用大红色做极简标记（如下划线或左竖线），无色块
- **WHEN** 用户查看表格
- **THEN** 表格极简风格：无背景色、无边框或极细灰边框、标签灰色
- **WHEN** 用户查看文章底部
- **THEN** 底部有一条细红装饰线，与顶部呼应
