# Hero 区域图片与内容层叠布局方案设计 Spec

## 现状分析

Hero 区域当前层叠结构：

```
z-index:0  — 装饰背景（::before, ::after, .hero-decoration）
z-index:2  — 轮播图片（.hero-carousel） ← 覆盖全区域
z-index:3  — 文字内容（.hero-content）     ← 左侧文字已修复 ✅
z-index:?  — 球体图（.hero-visual）        ← 无 z-index → 被图片遮挡 ❌
z-index:5  — 轮播指示点（.hero-carousel-dots）
```

**核心问题**：`.hero-visual`（包含 3 层同心圆轨道 + 节点条目）位于 `.hero-inner` 的正常流中，没有设置 `z-index`，在层叠上下文中被 `.hero-carousel{z-index:2}` 完全覆盖。

## 设计目标
1. 图片不遮挡左侧文字内容
2. 图片不遮挡右侧 3 个同心圆图形及其条目信息
3. 整体设计协调美观，保持品牌调性
4. 兼顾图片的可识别性和内容的可读性

## 设计方案评估

### 方案 A：半透明遮罩层（Overlay）

在图片层（z-index:2）和内容层（z-index:3）之间插入一个深色半透明遮罩层。

**实现方式**：
```css
.hero-overlay {
  position: absolute;
  top: 0; left: 0;
  width: 100%; height: 100%;
  background: linear-gradient(
    135deg,
    rgba(0, 0, 0, 0.55) 0%,    /* 左侧深色，确保文字可读 */
    rgba(171, 25, 66, 0.45) 40%, /* 中间过渡到品牌色 */
    rgba(171, 25, 66, 0.30) 100% /* 右侧浅色，球体区域 */
  );
  z-index: 2; /* 在图片之上，在内容之下 */
  pointer-events: none;
}
```

**透明度建议**：
| 区域 | 透明度 | 理由 |
|------|--------|------|
| 左侧（文字区域） | `rgba(0,0,0,0.55)` | 深色文字背景需要较高对比度 |
| 中间过渡 | 品牌色 45% | 柔和过渡 |
| 右侧（球体区域） | `rgba(171,25,66,0.30)` | 浅色遮罩，球体本身颜色明亮，不需要太深 |

**视觉影响**：
- 图片可见性：降低 30–55%，但仍可见轮廓和色调
- 内容可读性：显著提升，文字/球体清晰
- 空间利用率：不变（全幅覆盖）
- 用户体验：专业感强，常见于高端品牌站
- 响应式：CSS 实现，完全自适应

**优缺点**：
- ✅ 实现简单，一行 CSS
- ✅ 行业标准做法
- ✅ 内容可读性最佳
- ✅ 适配所有屏幕尺寸
- ❌ 图片本身被淡化
- ❌ 需要额外 DOM 元素

---

### 方案 B：仅修复球体 z-index（最小改动）

只给 `.hero-visual` 加上 `z-index:3`（与文字同级），图片保持全幅覆盖。

**实现方式**：
```css
.hero-visual {
  position: relative;
  z-index: 3; /* 与 .hero-content 同级，在图片之上 */
  display: flex;
  align-items: center;
  justify-content: center;
}
```

**层叠结构**：
```
z-index:2  — 轮播图片（全幅）
z-index:3  — 左侧文字 + 右侧球体
z-index:5  — 指示点
```

**视觉影响**：
- 图片可见性：无损失（全亮显示）
- 内容可读性：文字在图片上可读，但球体在图片上可能对比度不足
- 球体本身颜色较浅（`rgba(255,255,255,.15)` 背景 + 虚线边框），在深色品牌色渐变上原本就很明显，但在真实图片上可能不够清晰
- 空间利用率：最优（全幅图片）
- 用户体验：文字和球体都在图片之上，但球体的白色/透明元素在浅色图片上可能看不清

**优缺点**：
- ✅ 改动最小，仅加一行 CSS
- ✅ 不影响图片展示
- ✅ 无需额外 DOM/HTML 改动
- ❌ 球体在图片对比度不足时可能看不清
- ❌ 球体的白色半透明元素在浅色图片上会"消失"
- ❌ 图片亮度过高时文字阅览体验下降

---

### 方案 C：渐变遮罩 + 球体提亮（组合方案）

结合方案 A 和 B，使用右侧渐隐的遮罩 + 球体 z-index 提升。

**实现方式**：
```css
/* 遮罩层 — 左侧深色，右侧完全透明 */
.hero-overlay {
  position: absolute;
  top: 0; left: 0;
  width: 100%; height: 100%;
  background: linear-gradient(
    to right,
    rgba(0, 0, 0, 0.50) 0%,
    rgba(0, 0, 0, 0.15) 50%,
    transparent 70%
  );
  z-index: 2;
  pointer-events: none;
}

/* 球体提升 */
.hero-visual {
  position: relative;
  z-index: 3;
}
```

**视觉影响**：
- 左侧：遮罩使文字清晰可读
- 右侧渐变至透明：球体区域无遮罩，图片完整可见
- 球体在图片之上，靠自身的白色/品牌色元素显示
- 图片在右侧完全可见，视觉冲击力保持
- 空间利用率：最优

**优缺点**：
- ✅ 两侧分别优化（左侧遮罩 + 右侧图片全显）
- ✅ 球体在图片上通过自身亮度保持可见
- ✅ 专业感强，类似 Apple/Spotify 的 hero 处理方式
- ❌ 需额外 DOM 元素 + CSS 改动
- ❌ 球体的白色元素在亮色图片区域仍可能不够醒目

---

### 方案 D：图片位置偏移 + 右侧色块兜底

将图片容器偏移到仅覆盖左侧区域，右侧以品牌色渐变作为球体背景。

**实现方式**：
```css
.hero-carousel {
  position: absolute;
  top: 0; left: 0;
  width: 60%; /* 仅覆盖左侧 60% */
  height: 100%;
  z-index: 2;
  overflow: hidden;
}
.hero-slide-img {
  width: 100%;
  height: 100%;
  object-fit: cover;
  object-position: center right; /* 图片聚焦右侧裁剪边缘 */
}
```

**视觉影响**：
- 图片仅占左侧 60%，右侧 40% 是品牌色渐变
- 球体在纯净的品牌色背景上，极其清晰
- 文字区域有图片背景 + 可选遮罩
- 移动端可能需要调整比例或堆叠

**优缺点**：
- ✅ 球体区域背景纯净，完全不受图片干扰
- ✅ 球体可见性最佳
- ✅ 图片在左侧仍起到视觉冲击作用
- ❌ 右侧大块纯色区域可能显得空洞
- ❌ 响应式设计复杂（60% 比例在窄屏不适用）
- ❌ 需要调整图片裁切焦点
- ❌ 宽屏上右侧空档感更强

---

## 最终推荐：方案 C — 渐变遮罩 + 球体提亮

**选择理由**：

| 维度 | 方案 A | 方案 B | 方案 C | 方案 D |
|------|--------|--------|--------|--------|
| 改动量 | 中 | **小** | 中 | 大 |
| 文字可读性 | ★★★★★ | ★★★ | ★★★★★ | ★★★★★ |
| 球体可见性 | ★★★★ | ★★★ | ★★★★ | ★★★★★ |
| 图片表现力 | ★★★ | ★★★★★ | ★★★★ | ★★★ |
| 响应式适配 | ★★★★★ | ★★★★★ | ★★★★★ | ★★★ |
| 空间利用率 | ★★★★★ | ★★★★★ | ★★★★★ | ★★★ |

方案 C 在**文字可读性**（左侧遮罩）、**球体可见性**（右侧无遮罩 + z-index 提升）和**图片表现力**（右侧全亮图片）之间取得了最佳平衡。具体来说：

1. **左侧 0–50%**：渐深遮罩确保标题/描述在高对比度下清晰可读
2. **右侧 70–100%**：完全无遮罩，图片保持原始亮度和细节，球体自身靠半透明/品牌色在图片之上凸显
3. **过渡带 50–70%**：平滑渐变，无突兀边界
4. **改动量适中**：添加一个遮罩 DIV + 两行 z-index CSS

## Impact
- Affected specs: hj-image-system-final, fix-hero-text-overlay
- Affected code: `hj/public/index.html`

## 实施要点
- 遮罩层使用 `pointer-events: none` 确保不干扰点击
- 遮罩层位于 `.hero-carousel` 和 `.hero-content`/`.hero-visual` 之间
- 球体可见性如仍不足，可微调其元素亮度（如 `.diagram-core` 的 `box-shadow` 和 `border`）
