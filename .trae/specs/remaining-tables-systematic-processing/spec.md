# 剩余表格系统性处理（投资组合表 + 财务对比表）- 产品需求文档

## Overview
- **Summary**: 系统性处理股东信中剩余的约 100+ 个投资组合表和约 35 个财务年度对比表，统一转换为标准 Markdown 表格格式，确保数据准确、格式一致、可追溯。
- **Purpose**: 完成前一批次未处理的纯文本表格转换工作，实现股东信中所有主要表格的标准化呈现，提升阅读体验和数据可用性。
- **Target Users**: 深度阅读财务内容的投资者、研究员，以及依赖表格数据的所有读者。

## Goals
- 系统性处理所有剩余投资组合表（1996-2022 年约 30 个文件，以及 1965-1984 年约 15 个文件）
- 系统性处理财务年度对比表（约 30 个文件）
- 建立统一的表格处理规范和质量标准
- 确保数据 100% 准确，与原文完全一致
- 所有表格遵循统一的格式规范（表头、分隔符、空行、对齐等）

## Non-Goals (Out of Scope)
- 不处理 articles/、partnership/ 等其他目录的表格
- 不修改表格之外的任何正文内容
- 不涉及样式层面的修改（表格样式已标准化）
- 不重写或润色任何文字
- 不处理非表格的纯文本数据（如段落中的零散数字）

## Background & Context
- 前几轮已完成：表格重复表头修复、列数匹配、尾部空列清理、简单表格转换、股东信标题提取、损益表/资产负债表/收益来源表表格化、保险业务数据表（18个）、投资组合表第一批（1985-1995年，12个）
- 本轮剩余：投资组合表约 50+ 个（分布在约 40 个文件中），财务年度对比表约 30+ 个
- 统一标准：所有表格使用标准 Markdown 语法，前后有空行，列数一致

## Functional Requirements
- **FR-1**: 投资组合表转换为标准 Markdown 表格（股份数量、公司名称、成本、市值等列）
- **FR-2**: 财务年度对比表转换为标准 Markdown 表格（财务项目 × 年度）
- **FR-3**: 所有表格表头清晰，注明单位（千美元/百万美元/美元等）
- **FR-4**: 表格前后保留空行，符合 Markdown 规范
- **FR-5**: 特殊值（如"盈利"、"<0"、"预估"、"-"等）正确保留
- **FR-6**: 脚注标记和备注说明保留在表格附近

## Non-Functional Requirements
- **NFR-1**: 数据准确性——所有数字、文字与原文完全一致，零错误
- **NFR-2**: 格式一致性——所有表格遵循统一格式规范
- **NFR-3**: 可追溯性——所有修改可通过 Git 历史追溯和回滚
- **NFR-4**: 可验证性——每个表格都可以通过人工核对验证准确性

## Constraints
- **Technical**: 基于现有 Markdown 渲染管线，不引入新依赖
- **Business**: 内容必须忠实于原文，不得随意增删
- **Scope**: 仅处理 letters/ 目录下的股东信文件
- **Quality**: 数据准确性优先于处理速度

## Assumptions
- 投资组合表的结构在不同年份之间有延续性，可以参考已处理的年份
- 财务年度对比表主要分为两类：横向（年度为列）和纵向（年度为行）
- 后期年份（2000年后）的投资组合表可能列数更多（如有持股比例、市值变动等）
- 有些表格可能已经是 Markdown 格式（需跳过）

## Acceptance Criteria

### AC-1: 投资组合表全部表格化
- **Given**: 股东信中以纯文本形式存在的投资组合持仓表
- **When**: 转换为标准 Markdown 表格后
- **Then**: 表格列结构正确，数据完整，对齐一致
- **Verification**: `human-judgment`
- **Notes**: 包括普通股、优先股、可转债等不同类型的持仓表

### AC-2: 财务年度对比表全部表格化
- **Given**: 股东信中以纯文本形式存在的多年份财务对比表
- **When**: 转换为标准 Markdown 表格后
- **Then**: 财务项目名称正确，各年度数据准确，结构清晰
- **Verification**: `human-judgment`

### AC-3: 数据准确性
- **Given**: 转换后的表格
- **When**: 与原文逐字逐句核对
- **Then**: 所有数字、文字、符号、格式完全一致，零错误
- **Verification**: `human-judgment`

### AC-4: 格式一致性
- **Given**: 所有转换后的表格
- **When**: 检查格式规范
- **Then**: 表格前后有空行，表头格式统一，列数一致，分隔符行正确
- **Verification**: `programmatic` + `human-judgment`

### AC-5: 浏览器端正确渲染
- **Given**: 修复后的内容页面
- **When**: 在浏览器中访问
- **Then**: 所有表格以表格形式正确渲染，支持暗色模式和移动端滚动
- **Verification**: `human-judgment`
